Need of oops:

So if oops was not there then we would have simply been doing procedural programming.
A kind of programming in which there is no classes and objects just simple functions and data varibales, directly
getting accessed and changed etc.

The problem with procedural programming is that it does not allow us to write something reusable throughout
the program and thus can give rise to code duplicacy and moreover there is no data security and data is being
used and modified by multiple procedures and thus we know there is a problem.

But what is the solution. The solution is Oops i,e object oriented programming.


So in object orinted programming we have a 

- Class : A class is like a blueprint that defines the properties and behaviors of objects.

- Object: is something which is an instance of a class and follows the blueprint of its class.

- Data members: Its also called as the state of the class for example lets say we are implementing a light bulb using 
programming so a light bulb can have two staes i,e on/off and it can have two behaviours as well which is it can be turned
on and it can be turned off. So the states is refered as datamembers.

-Methods: In the above example of the behaviours of the bulb it can have two behaviours i,e it can be turned on and 
it can be turned off. Such behaviours in oops can be implemented or being refered as methods.
